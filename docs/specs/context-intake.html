Context Intake Job Spec

Capture completed Omi conversations into local, source-backed Foundation intake records and draft review notes without committing raw private payloads.

Status: active
Updated: 2026-06-15
Parent: foundation.knowledge-data-layer.system

Intent

This job makes conversation intake repeatable: Omi sends a processed memory payload, the Mac mini stores the source locally, and Foundation receives a draft note that can be reviewed before it becomes substrate.

Trigger

The job starts when Omi calls the configured Memory Creation webhook for a completed conversation.

Capability supported

CapabilityHow this job supports itEvidence or source
foundation.knowledge-data-layer.systemTurns ambient conversation data into durable context candidates with local source evidence.Omi webhook payloads, SQLite rows, and reviewed Markdown notes.

Current handling

  • Current actor: human operator and ad hoc agent conversations.
  • Current path: conversation context may remain in chat history or manual notes.
  • Known gaps: no stable source id, no local raw retention, no dedupe, no review note template, and no webhook intake path.

Proposed handling

  • Proposed actor: Omi webhook, local FastAPI service, SQLite, note generator, and reviewing human or agent.
  • Proposed path: accept authenticated memory webhook, persist raw and normalized records locally, generate a draft note, then review before committing.
  • Decision state: accepted for v1.

Process

  1. Prepare: run the Mac mini service, Cloudflare Tunnel, SQLite runtime directory, and Omi memory webhook configuration.
  2. Validate: reject webhook calls with a missing or incorrect shared token.
  3. Record: write raw JSON to ignored local storage and normalize source, conversation, transcript, and note rows into SQLite.
  4. Draft: render a Markdown note with summary, action items, candidate signals, excerpts, and review checklist.
  5. Review: remove sensitive material, mark useful signals, and change note status to reviewed only when safe.
  6. Promote: use existing Foundation spec workflows for any reviewed note that should become durable capability, job, process, decision, or evidence substrate.
