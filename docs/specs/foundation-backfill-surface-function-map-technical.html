Foundation Backfill Surface / Function Map Technical Spec

This technical layer makes Foundation Backfill Surface / Function Map deterministic enough for checks and structured enough for agent judgment.

Status: active
Updated: 2026-05-28
Parent: foundation.backfill-surface-function-map.descriptive

Implementation must preserve the source-of-record split: JSONL is canonical for layer rows and eval receipts, while HTML summaries are generated for human reading. Each command must be resumable from artifacts and must not rely on chat memory. Surface / Function Map is scoped to files that can define capability surfaces; Artifact Inventory remains the exhaustive artifact ledger.

Contracts

ContractRuleReason
Upstream completenessThe layer starts only from a passing upstream handoff or an explicit blocker.Later synthesis must not hide earlier discovery gaps.
Canonical JSONLLayer rows and eval receipts are JSONL; HTML summaries are derived.Agents need durable machine state and users need readable review surfaces.
One-file fill loopAgents process exactly one eligible upstream file per mark command: select next target, read the complete file, mark every surface for that file with inline JSON and concrete evidence, then move on. The fill command rejects generated surface files, batch options, all-file fill modes, missing evidence, and generic evidence that only says the file was read. Ineligible rows never enter the queue.Surface rows come from source review rather than path, Artifact Inventory metadata, grep output, generated summaries, or helper-script inference, while inert artifacts do not consume context.
Deterministic checkervalidates upstream Artifact Inventory check/eval handoff, every Surface / Function Map-eligible file row represented by at least one surface row/support classification/review blocker, no surface rows attached to ineligible files, concrete full-file evidence, surface kind/source/status enums, upstream references, duplicate surface IDs, required fields, stale upstream hashes, handoff states, and optional report drift.Structural drift should fail mechanically before semantic evaluation.
Semantic evalsamples surface rows against Artifact Inventory rows to judge whether exposed and dependent surfaces were missed, over-merged, mislabeled, weakly supported, vague, or detached from full-file evidence.Correctness includes meaning, specificity, and evidence quality.
RefreshWhen upstream Artifact Inventory rows change, matching Surface / Function Map rows are preserved only if stored upstream hashes still match. Changed/new upstream rows return to pending and must be read and marked again one file at a time.The layer participates in steady-state changed-file refresh instead of only bootstrap backfill.
Handoff gateall Surface / Function Map-eligible file rows are resolved to surfaces or explicit support/review classifications; skipped Artifact Inventory rows are counted as outside scope; checker passes; eval receipt and HTML summary exist with zero revision targets; report names next layer as Capability Map only after warnings are resolved.Each layer is a reproducible correct unit before the next layer consumes it.

Checker and fill loop

  1. Read active report, run log, upstream Artifact Inventory artifacts, and current Surface / Function Map artifact.
  2. Select the next pending or failed eligible upstream file with foundation:surface-function-map:fill -- --next.
  3. Read the complete eligible source file. Do not classify from path, Artifact Inventory kind, imports, grep output, generated summaries, helper-script output, or any summary without full-file reading. Do not read skipped artifacts just to prove they are skipped.
  4. Write every surface for that one file with foundation:surface-function-map:fill -- --path <file> --surfaces-json .... The command replaces prior rows for only that upstream file and records concrete agent-read-full-file evidence naming handlers, resources, tables, commands, dependencies, rules, or spec sections. It must not accept --surfaces-file, --all, --batch-size, missing evidence, generic read-only evidence, or equivalent bulk-generation paths.
  5. Run the checker; repair structural failures before semantic eval.
  6. Run or append eval receipts; generate HTML summary from JSONL.
  7. Revise every row named in eval revisionTargets, rerun eval, and record report state with nextLayer set to Capability Map only after check and eval pass with zero revision targets.
  8. Revise failed rows until the gate passes or a named human decision blocks progress.
