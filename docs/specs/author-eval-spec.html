Author Eval Spec Job

This job authors or revises eval specs after Spec Selection classifies the work as evidence judgment.

Status: active
Updated: 2026-06-06
Canonical section: #job-intent

Job intent

This job turns a Spec Selection Result for evaluation into an HTML-native eval spec that defines evidence, acceptance, failure, judge method, revision targets, and gaps.

Capability supported

Supports foundation.author-durable-specs.capability by making quality gates durable, source-backed, and reusable across agent runs.

Current handling

Without a dedicated eval lane, agents can treat tests, scorecards, judge prompts, and review notes as interchangeable. This lane preserves the distinction: tests collect evidence; evals judge whether evidence is good enough.

Proposed handling

The lane starts only from a Spec Selection Result whose primary classification is eval or that names an eval companion spec as required. It must attach to the spec or workflow being judged.

Process

  1. Read the Spec Selection Result, target spec, eval template, and AI eval principles when a judge or rubric is involved.
  2. Run the readiness gate: ask only questions that materially change evidence, pass/fail/block conditions, judge method, revision routing, or anti-shortcut cases; otherwise continue with labeled assumptions.
  3. Define what evidence is collected, how it is judged, what fails, what revisions are routed, and what gaps remain.
  4. Add anti-shortcut cases when agents could pass structurally while missing intent.
  5. Author or revise HTML prose, metadata, graph edges, coverage, and visible links.
  6. Run the eval-spec eval, revise blocking findings, then run registry and graph checks.

Context, tools, and interfaces

  • skills/author-eval-spec/SKILL.md is the agent procedure.
  • docs/specs/templates/eval-spec-template.html is the authoring shape.
  • foundation.author-eval-spec.eval is the quality gate.

Readiness questions: What evidence should be judged? What must pass, fail, block, or route revision? What judge method or deterministic check should be used? What shortcuts or generic outputs must this eval reject?

Evidence and evaluation

The authored eval spec is acceptable only when evidence sources, pass/fail conditions, judge method, revision targets, blocker semantics, and fixture/calibration needs are explicit.

Revision

Revise this job when agents accept vague quality language, fail to name revision targets, confuse tests with evals, or omit anti-shortcut cases for agent output.
