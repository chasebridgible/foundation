Foundation Backfill Split And Queue Technical Spec

This technical layer makes Foundation Backfill Split And Queue deterministic enough for scripts and structured enough for agent judgment.

Status: active
Updated: 2026-05-29
Parent: foundation.backfill-split-queue.descriptive

Implementation must preserve the source-of-record split: JSONL is canonical for layer rows and eval receipts, while HTML summaries are generated for human reading. Each command must be resumable from artifacts and must not rely on chat memory.

Contracts

ContractRuleReason
Upstream completenessThe layer starts only from a passing upstream handoff or an explicit blocker.Later synthesis must not hide earlier discovery gaps.
Canonical JSONLLayer rows and eval receipts are JSONL; HTML summaries are derived.Agents need durable machine state and users need readable review surfaces.
Bounded fill loopAgents process rows or slices in small batches and run checks after each batch.Long-running work survives context compaction and handoff.
Deterministic checkervalidates every terminal capability has queue coverage, every slice references valid and current capability IDs, needs-split capabilities have multiple child slices or an explicit blocker, terminal statuses require evidence, and active slices carry exactly one current next action.Structural drift should fail mechanically before semantic evaluation.
Semantic evalsamples slices against capability rows to judge whether the slice is small enough, exit criteria are objective, evidence references are specific, and no hidden behavior is parked inside a parent summary. Findings at warning or blocking severity become revision targets.Correctness includes meaning, specificity, and evidence quality.
Handoff gatedurable queue is complete, no needs-split row lacks child slices, checker passes, eval receipt and HTML summary exist, and report names next layer as Evidence Pack.Each layer is a reproducible correct unit before the next layer consumes it.

Checker and fill loop

  1. Read active report, run log, upstream artifacts, and current layer artifact.
  2. Select next pending or failed batch by deterministic order plus risk priority.
  3. Inspect only the evidence needed for the batch and update canonical JSONL rows.
  4. Run the checker; repair structural failures before semantic eval.
  5. Run or append eval receipts; generate HTML summary from JSONL.
  6. Revise failed rows until the gate passes or a named human decision blocks progress.
