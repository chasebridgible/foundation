Foundation Backfill Handoff Technical Spec

This technical layer makes Foundation Backfill Handoff deterministic enough for scripts and structured enough for agent judgment.

Status: draft
Updated: 2026-06-07
Parent: foundation.backfill-handoff.job

Implementation must preserve the source-of-record split: JSONL is canonical for layer rows and eval receipts, while HTML summaries are generated for human reading. Each command must be resumable from artifacts and must not rely on chat memory.

Data model

ObjectRequired fieldsInvariant
Layer rowrunId, rowId, upstreamIds, status, confidence, evidenceRefs, reviewFlags, createdAt, updatedAtEvery row has stable identity and traceable upstream evidence.
Canonical artifacthandoff-report-YYYYMMDD-NN.jsonl, handoff-eval-YYYYMMDD-NN.jsonlJSONL files are append-friendly and machine-parseable; each line is one receipt or row.
Derived artifacthandoff-summary-YYYYMMDD-NN.html, final review reportDerived views can be regenerated and never replace canonical JSONL state.
Report entryartifact paths, row counts, pending counts, checker result, eval result, latest run-log sequence, next layerA new agent can resume from report plus artifacts.

Contracts

ContractRuleReason
Upstream completenessThe layer starts only from a passing upstream handoff or an explicit blocker.Later synthesis must not hide earlier discovery gaps.
Canonical JSONLLayer rows and eval receipts are JSONL; HTML summaries are derived.Agents need durable machine state and users need readable review surfaces.
Handoff gate loopAgents produce one final run report from accepted graph state, fail handoff when parent-only, needs-split, blocked-without-decision, stale, or under-reviewed rows remain, and rerun handoff after revisions.Long-running work survives context compaction and prevents final-report rationalization.
Deterministic checkervalidates every upstream layer has a terminal receipt, artifact paths resolve, run-log sequence is current, blockers are named, and report state does not claim completion with unresolved structural failures.Structural drift should fail mechanically before semantic evaluation.
Semantic evaljudges whether the handoff lets the user decide what to trust, what to review, what to implement next, and what remains uncertain without replaying the entire run.Correctness includes meaning, specificity, and evidence quality.
Handoff gatefinal report is ready for human review; all layer receipts are linked; no hidden pending work remains outside explicit blockers or review questions.Each layer is a reproducible correct unit before the next layer consumes it.

Checker and fill loop

  1. Read active report, run log, upstream artifacts, and current layer artifact.
  2. Select next pending or failed batch by deterministic order plus risk priority.
  3. Inspect only the evidence needed for the batch and update canonical JSONL rows.
  4. Run the checker; repair structural failures before semantic eval.
  5. Run or append eval receipts; generate HTML summary from JSONL.
  6. Revise failed rows until the gate passes or a named human decision blocks progress.

Failure modes

FailureSystem behaviorEvidence
Missing upstream rowChecker fails and names the missing upstream ID.Checker JSON output and report entry.
Vague or unsupported rowEval routes the row back to the fill loop.Eval JSONL receipt with revision target.
Report driftChecker fails when report paths/counts do not match artifacts.Report check result.
Context resetNext agent resumes from report, run log, current artifacts, and pending rows.Run-log checkpoint.
