Agent Capability Scout

The Agent Capability Scout wakes every other day at 5am, researches bounded agent-system sources, records what changed in Foundation JSON/JSONL artifacts, assigns each meaningful finding a 1-10 interest grade, drafts principles-doc patches for durable lessons, and notifies the owner through GitHub review comments when the run is complete.

Intent

Make external agent-system learning reliable enough to compound without relying on model memory, unmerged notes, or unbounded research.

Trigger

A Codex automation run every other day at 5am, a manual owner request, or a source-registry refresh.

Current handling

  • Current actor: Human-led ad hoc research and chat discussion.
  • Current path: The owner asks what is available, reads summaries, and decides manually whether a new agent capability matters.
  • Known gaps: No scheduled wake-up, no bounded source registry, no observed-diff archive, no repeatable interest grading, no principle-candidate path, no GitHub notification receipt, and no durable Foundation artifact update path.

Proposed handling

  • Proposed actor: Codex automation plus LLM summarizer, operating in a clean Foundation worktree and closing durable updates through Foundation GitHub.
  • Proposed path: Start a clean branch or worktree, fetch bounded sources, record snapshots or diffs, normalize findings, assign 1-10 interest grades, identify durable principle candidates, patch the relevant principles docs when warranted, update JSON/JSONL artifacts, run checks, push through Foundation's protected flow, and notify the owner through a GitHub comment with optional email fallback.
  • Reason for change: Agent capabilities change quickly, but unmanaged research can waste founder time or leave useful learning outside durable Foundation memory.
  • Decision state: proposed.

Process

  1. Wake: Codex automation starts every other day at 5am with run ID, source allowlist, prior run state, grading rubric, principle-candidate rubric, and notification target.
  2. Prepare: create or enter a clean Foundation branch/worktree so durable artifacts and docs can be committed without mixing unrelated user changes.
  3. Observe: fetch only allowlisted sources for OpenAI, Anthropic, Google, and Addy Osmani agent-related material; store raw snapshots or observed diffs with timestamps and source metadata.
  4. Normalize: convert relevant source changes into structured finding records with source, date seen, change type, summary, evidence path, confidence, and 1-10 interest grade.
  5. Grade: assign interest or value based on broad potential to improve agent systems; a grade is a triage signal, not an approval state.
  6. Promote candidates: when a finding looks durable across tools, vendors, or models, create a principle-candidate record that names the target principles document, proposed wording or patch path, evidence, and why it belongs in doctrine rather than only in the findings log.
  7. Write: append the canonical JSONL run log, findings artifact, and principle-candidates artifact; update any derived brief or index; and avoid creating experiment plans unless the owner asks later.
  8. Validate and publish: run relevant Foundation checks, commit the artifact and principles-doc updates, push a branch or PR, and merge only when the protected Foundation flow and review expectations allow it.
  9. Notify: create a GitHub issue or PR comment with run status, artifact or PR path, top findings, interest grades, principle candidates, blockers, and next action; send email only if configured as a secondary or fallback channel.

Revision

  • Revision target: source registry, data schemas, interest grading rubric, principle-candidate rubric, notification adapter, merge closeout, or eval spec.
  • Update trigger: repeated low-value briefs, missed important source changes, unverifiable summaries, noisy grades, weak principle candidates, notification failure, or failed Foundation merge closeout.
  • Promotion rule: a proposed scout implementation becomes current handling only after a run fixture with JSON/JSONL artifacts, interest grades, principle-candidate state, merge state, and GitHub notification satisfies the eval spec.
