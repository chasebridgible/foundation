Agent Capability Scout Eval Spec

If this eval passes, a reviewer can trust that the scout looked only where it was allowed to look, preserved evidence for what it saw, separated no-change from meaningful findings, graded interest from 1 to 10, proposed principles-doc patches only for durable lessons, updated durable Foundation artifacts, and notified the owner with enough context to review the run.

Acceptance conditions mapped to specs

AC-1Every run records wake-up metadata and terminal status.foundation.agent-capability-scout.job#processRun manifest has every-other-day 5am trigger, timestamps, source registry version, and complete/blocked/failed status.

AC-6Principle candidates are evidence-backed, durable, standalone, and additive.foundation.agent-capability-scout.technical#data-modelEach candidate names target doc, proposed wording or patch path, source evidence, standalone eval result, additive rationale, and why it belongs in principles rather than only a findings log.
AC-7Durable memory closes through Foundation GitHub.foundation.agent-capability-scout.technical#internal-sequenceRun has merge receipt or explicit blocker naming branch, commit, PR, check, or permission state; clean detached worktrees create a dated scout branch before artifact edits instead of stopping in review-mode.

Commands or evidence required

  • npm run spec:check - proves spec registry, graph metadata, and Foundation self-map resolve after scout specs are added.
  • npm run foundation:agent-capability-scout:check - validates live scout artifact shape.
  • npm run foundation:agent-capability-scout:test - validates scout schema, fixture, grade, principle-candidate, merge, and notification checks.
  • docs/specs/agent-capability-scout/findings.jsonl - canonical finding artifact with evidence and 1-10 interest grades.
  • docs/specs/agent-capability-scout/principle-candidates.jsonl - canonical principle-candidate artifact with target doc, proposed wording or patch path, evidence, standalone eval, additive rationale, and durability rationale.
  • docs/specs/agent-capability-scout/briefs/YYYY-MM-DD-agent-capability-scout.md - live output that a human can inspect.
  • docs/specs/agent-capability-scout/notifications.jsonl - receipt showing the owner was notified by GitHub comment and optionally email.

Co-evolution rule

Required: when the scout job changes scope, source families, artifact schemas, interest grading, principle-candidate promotion, merge closeout, or notification requirements, this eval spec and the technical spec change in the same branch.

During review, check whether the job's process, technical data model, and eval acceptance conditions still describe the same state machine. If one changed without the others, the diff is incomplete unless the review evidence explains why verification did not change.
