Foundation Backfill Handoff

Handoff produces the final reviewable report for a backfill run: covered layers, artifact paths, accepted slices, blockers, uncertainties, stale docs, draft versus ready specs, and next implementation or review decisions. This layer answers: What was covered, what remains uncertain, and what should a human or next agent do?

Status: draft
Updated: 2026-05-26
Upstream: Evaluate System Coherence
Next: Human Review

This layer is a reproducible unit in the backfill substrate. It initializes from the previous layer, fills bounded batches, checks deterministic structure, writes canonical JSONL eval receipts, derives a readable HTML summary, revises failed rows, and records handoff state before the next layer begins.

GBrain's source-of-record lesson applies directly: canonical append-friendly artifacts own state; readable HTML summaries and report tables are derived views. Later agents cite row IDs and evidence links instead of relying on inherited conversation memory.

Small-loop journey

  1. Initialize: create the canonical Foundation Backfill Handoff artifact from Evaluate System Coherence handoff output.
  2. Select batch: choose pending rows or slices using deterministic order plus risk priority.
  3. Fill: agent compiles layer receipts, checker/eval status, accepted slices, unresolved human questions, stale/conflicting evidence, draft specs, ready specs, and next recommended Foundation work.
  4. Check: run the deterministic checker and record result in the active report and run log.
  5. Evaluate: write canonical JSONL eval receipts and a readable HTML summary.
  6. Revise: route failed rows back to the fill loop with row IDs and findings.
  7. Handoff: advance only when the handoff gate passes or named human decisions block the layer.

Checker and eval contract

MechanismContract
Checkervalidates every upstream layer has a terminal receipt, artifact paths resolve, run-log sequence is current, blockers are named, and report state does not claim completion with unresolved structural failures.
Eval receiptjudges whether the handoff lets the user decide what to trust, what to review, what to implement next, and what remains uncertain without replaying the entire run. Eval output is canonical JSONL; HTML is a derived summary for human review.
Sandia Oil calibrationThe first real calibration target is Sandia Oil. Its output is not a gold benchmark unless a human-reviewed expected Artifact Inventory is created; it is used to catch obvious misses while context is tractable.

Handoff gate

final report is ready for human review; all layer receipts are linked; no hidden pending work remains outside explicit blockers or review questions.

  • Canonical artifacts exist and share a run ID.
  • All rows are in valid terminal or explicitly blocked states.
  • Checker passes for structure, references, currentness, and required fields.
  • Eval JSONL and HTML summary exist or an explicit planned-gap risk is recorded.
  • Active report records counts, artifact paths, checker/eval status, run-log sequence, and next layer.
