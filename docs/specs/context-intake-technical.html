Context Intake Technical Spec

Define the local FastAPI, SQLite, raw-payload, note artifact, and Omi webhook contracts for v1 context intake.

Status: active
Parent: foundation.context-intake.job
Updated: 2026-06-15

Depth calibration

This is deeper than a small script because it handles private conversation data, public HTTPS webhook exposure, local persistence, and repo artifact boundaries.

Non-goals

V1 does not run local Whisper, ingest raw audio bytes, promote notes into specs automatically, or expose a public admin UI.

Internal sequence

  1. Trigger: Omi POSTs a Memory Creation payload to /webhooks/omi/memory/{webhook_token}.
  2. Validate: the path token must match CONTEXT_INTAKE_WEBHOOK_TOKEN.
  3. Persist raw: write canonical JSON under context-intake/runtime/raw/YYYY/MM/DD/.
  4. Normalize: upsert source, conversation, transcript, and note artifact rows into SQLite.
  5. Draft note: write a deterministic Markdown note unless an existing note would be overwritten.
  6. Observe: return source id, conversation id, note path, and note creation status.

Contracts and integrations

  • GET /health returns path and token readiness.
  • POST /webhooks/omi/memory/{webhook_token} accepts Omi memory webhook JSON and returns source id, conversation id, note path, and note creation status.
  • GET /admin/intakes?token=... lists recent intakes for local administration.
  • POST /admin/intakes/{conversation_id}/regenerate-note?token=... rewrites a note from the latest raw payload.
  • context-intake/bin/run-server.sh starts the API from the repo venv and ignored local env file.
  • context-intake/launchd/com.foundation.context-intake.plist keeps the local API running across user sessions.
