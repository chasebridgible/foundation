Foundation Backfill Capability Map

Capability Map converts surfaces into actor/outcome units using the formula: actor, intended outcome, domain object, actions, states, rules, experience, backing contracts, failure/recovery, and evidence. This layer answers: What is this system supposed to let someone or something do?

Status: active
Updated: 2026-05-29
Upstream: Surface / Function Map
Next: Define Spec Jobs

This layer is a reproducible unit in the backfill substrate. It initializes from the previous layer, fills reviewed surface groups, checks deterministic structure, writes canonical JSONL eval receipts, derives a readable HTML summary, revises failed rows, and records handoff state before the next layer begins.

GBrain's source-of-record lesson applies directly: canonical append-friendly artifacts own state; readable HTML summaries and report tables are derived views. Later agents cite row IDs and evidence links instead of relying on inherited conversation memory.

Small-loop journey

  1. Initialize: create the canonical Foundation Backfill Capability Map artifact from Surface / Function Map handoff output.
  2. Select target: choose the next pending or failed ready surface using deterministic order plus review priority.
  3. Fill: agent reviews the selected surface and any nearby ready surfaces, groups only reviewed surfaces into capability rows, and names actor, outcome, domain object, actions, states, rules, experience, backing contracts, failure/recovery behavior, evidence rows, confidence, and review flags.
  4. Check: run the deterministic checker and record result in the active report and run log.
  5. Evaluate: write canonical JSONL eval receipts and a readable HTML summary.
  6. Revise: route failed rows back to the fill loop with row IDs and findings.
  7. Handoff: advance only when the handoff gate passes or named human decisions block the layer.

Checker and eval contract

MechanismContract
Checkervalidates the upstream Surface / Function Map handoff, capability JSONL parsing, every ready surface maps to a terminal capability row, every capability has the full formula fields, broad rows are marked needs-split, upstream surface freshness refs match, evidence references resolve, and status values are valid.
Eval receiptsamples capabilities against surfaces and source evidence to detect vague outcomes, missing actors, missing state/rule contracts, over-merged domains, and unsupported capability claims. Eval output is canonical JSONL; HTML is a derived summary for human review.
Sandia Oil calibrationThe first real calibration target is Sandia Oil. Its output is not a gold benchmark unless a human-reviewed expected Artifact Inventory is created; it is used to catch obvious misses while context is tractable.

Handoff gate

all ready Surface / Function Map rows are capability-owned, broad rows are flagged for split, checker passes, eval receipt and HTML summary exist with zero revision targets, and report names next layer as Define Spec Jobs.

  • Canonical artifacts exist and share a run ID.
  • All rows are in valid terminal states: ready-for-queue or needs-split.
  • Checker passes for structure, references, currentness, and required fields.
  • Eval JSONL and HTML summary exist and list zero revision targets.
  • Active report records counts, artifact paths, checker/eval status, run-log sequence, embedded backfill-capability-map JSON, and next layer.
