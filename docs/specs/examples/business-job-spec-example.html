Pay employees accurately and on time

This job spec describes the body of work required for a business to run payroll for one pay period without late pay, incorrect pay, missing approvals, or unrecorded exceptions.

Status: active example
Type: job
Updated: 2026-05-31

Capability supported

Pay employees accurately and on time is a capability because it names a reliable outcome the business must be able to produce. The payroll run is a job inside that capability.

Current handling to capture

A real client version would describe how payroll works today: who gathers hours, which system holds time data, who approves exceptions, which vendor submits payroll, what errors happen, and what evidence proves the run was accepted.

Proposed handling

If the payroll job is being improved, the proposal stays here: changed actor, changed approval path, new checklist, agent-assisted exception review, new data capture, or a different payroll tool. When accepted, this becomes current handling and the old version moves into revision history.

Process inside the job

  1. Gather pay-period inputs. Pull approved hours, PTO, commissions, deductions, new hires, terminations, and pay-rate changes.
  2. Review exceptions. Resolve missing punches, unapproved overtime, employee status changes, and manager disputes before submission.
  3. Calculate payroll. Use the payroll system or provider workflow to calculate gross pay, taxes, deductions, and net pay.
  4. Approve and submit. The accountable actor approves payroll before the submission deadline.
  5. Record evidence. Save confirmation, payroll register, exception log, and any follow-up changes.
  6. Revise the substrate. If the run exposed a recurring issue, update the job spec, decision rule, source registry, or eval spec.
