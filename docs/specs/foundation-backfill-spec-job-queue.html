Foundation Backfill Define Spec Jobs

Define Spec Jobs decides whether capability rows are too broad and creates the durable work queue that prevents agents from closing coarse slices prematurely. This layer answers: What exact slice should the agent work on next, and when is it complete?

Status: active
Updated: 2026-06-07
Upstream: Capability Map
Next: Context Pack

This layer is a reproducible unit in the backfill substrate. It initializes from the previous layer, fills bounded batches, checks deterministic structure, writes canonical JSONL eval receipts, derives a readable HTML summary, revises failed rows, and records handoff state before the next layer begins.

GBrain's source-of-record lesson applies directly: canonical append-friendly artifacts own state; readable HTML summaries and report tables are derived views. Later agents cite row IDs and evidence links instead of relying on inherited conversation memory.

Small-loop journey

  1. Initialize: create the canonical Foundation Backfill Define Spec Jobs artifact from Capability Map handoff output.
  2. Semantic audit: before queue rows can be accepted, verify the upstream Capability Map has no unresolved needs-split rows, parent rows have child outcomes, and every queue source is a queue-eligible child or sole capability.
  3. Select batch: choose pending child/sole rows or slices using deterministic order plus risk priority.
  4. Fill: agent writes exact exit criteria, owner spec targets, next action, and blocking questions for one precise child/sole capability slice without hiding multiple behavior families in one row.
  5. Check: run the deterministic checker and record result in the active report and run log.
  6. Evaluate: write canonical JSONL eval receipts and a readable HTML summary.
  7. Revise: route failed rows back to the fill loop with row IDs and findings.
  8. Handoff: advance only when the handoff gate passes or named human decisions block the layer.

Checker and eval contract

MechanismContract
Checkervalidates every queue-eligible child/sole capability has queue coverage, every slice references valid and current child/sole capability IDs and refs, parent, needs-split, blocked, and non-queueable rows cannot enter the queue, terminal statuses require evidence, active slices carry exactly one current next action, child/sole slices align to capability identity rather than incidental source paths, and any existing eval receipt matches the current queue fingerprint.
Eval receiptsamples slices against capability rows to judge whether the slice is small enough, aligned to the capability identity, matched to splitCriteria, exit criteria are objective, evidence is specific, and no hidden behavior is parked inside a parent summary. Warnings produce revision targets; report handoff is blocked until those targets are resolved.
Sandia Oil calibrationThe first real calibration target is Sandia Oil. Its output is not a gold benchmark unless a human-reviewed expected Artifact Inventory is created; it is used to catch obvious misses while context is tractable.

Handoff gate

Job / Spec Queue is complete, every queued slice references only queue-eligible child/sole capabilities, checker passes, eval receipt and HTML summary exist for the exact current queue artifact, and report names next layer as Context Pack.

  • Canonical artifacts exist and share a run ID.
  • All rows are in valid terminal or explicitly blocked states.
  • Checker passes for structure, references, currentness, and required fields.
  • Eval JSONL and HTML summary exist, carry the current queue fingerprint, and have zero revision targets.
  • Active report records counts, artifact paths, checker/eval status, run-log sequence, and next layer.
