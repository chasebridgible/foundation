Intent

State the product promise and the user-visible result. Avoid implementation nouns unless the user would naturally say them.

Out of scope

List adjacent behaviors this spec does not cover. Link child specs instead of expanding into every lower-level detail.

User model

Define who uses this, what they are trying to accomplish, what they know before entering the flow, and what language they would use. Use product/user language, not implementation language.

UserGoalContextSuccess feeling
[primary user][plain-English goal][entry context][what good feels like]

Interface journey

Describe the actual journey in order. Include entry points, visible controls, expected feedback, transitions, and exit states.

  1. Entry: how the user gets here and what they see first.
  2. Primary action: what the user does, what control they use, and what happens immediately.
  3. System response: what changes on screen, what stays stable, and what the user can do next.
  4. Exit: what counts as a finished interaction and where the user lands.
