Foundation Backfill Surface Registry

Surface Registry converts mapped file rows into evidence surfaces: routes, screens, APIs, commands, jobs, tables, workflows, infra resources, docs, tests, generated artifacts, and external dependencies. This layer answers: What things does this repo expose or depend on?

Status: draft
Updated: 2026-05-26
Upstream: File Registry
Next: Capability Matrix

This layer is a reproducible unit in the backfill substrate. It initializes from the previous layer, fills bounded batches, checks deterministic structure, writes canonical JSONL eval receipts, derives a readable HTML summary, revises failed rows, and records handoff state before the next layer begins.

GBrain's source-of-record lesson applies directly: canonical append-friendly artifacts own state; readable HTML summaries and report tables are derived views. Later agents cite row IDs and evidence links instead of relying on inherited conversation memory.

Small-loop journey

  1. Initialize: create the canonical Foundation Backfill Surface Registry artifact from File Registry handoff output.
  2. Select batch: choose pending rows or slices using deterministic order plus risk priority.
  3. Fill: agent selects mapped file rows by risk priority, extracts concrete surfaces, records upstream file row IDs, evidence paths, surface kind, actor or consumer hints, operations, state/rule hints, and review flags.
  4. Check: run the deterministic checker and record result in the active report and run log.
  5. Evaluate: write canonical JSONL eval receipts and a readable HTML summary.
  6. Revise: route failed rows back to the fill loop with row IDs and findings.
  7. Handoff: advance only when the handoff gate passes or named human decisions block the layer.

Checker and eval contract

MechanismContract
Checkervalidates every behavior-bearing file row is represented by at least one surface row, intentionally classified as non-surface support, or flagged for review; validates surface kind enums, upstream file row references, duplicate surface IDs, and current upstream hashes.
Eval receiptsamples surface rows against file registry rows and source snippets to judge whether exposed and dependent surfaces were missed, over-merged, mislabeled, or detached from evidence. Eval output is canonical JSONL; HTML is a derived summary for human review.
Sandia Oil calibrationThe first real calibration target is Sandia Oil. Its output is not a gold benchmark unless a human-reviewed expected registry is created; it is used to catch obvious misses while context is tractable.

Handoff gate

all behavior-bearing file rows are resolved to surfaces or explicit support/review classifications; checker passes; eval receipt and HTML summary exist; report names next layer as Capability Matrix.

  • Canonical artifacts exist and share a run ID.
  • All rows are in valid terminal or explicitly blocked states.
  • Checker passes for structure, references, currentness, and required fields.
  • Eval JSONL and HTML summary exist or an explicit planned-gap risk is recorded.
  • Active report records counts, artifact paths, checker/eval status, run-log sequence, and next layer.
