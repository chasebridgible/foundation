Foundation Backfill Evaluate Job Slices Eval Spec

This test layer defines the deterministic and semantic gates that prove Foundation Backfill Evaluate Job Slices can hand off safely.

Status: draft
Updated: 2026-05-26
Parent: foundation.backfill-job-slice-evaluation.job

The tests treat the layer as a reproducible unit: structure is checked mechanically, semantic quality is judged through eval receipts, and the HTML summary is a derived review surface.

Acceptance mapping

IDConditionMust prove
AC-1Canonical artifacts exist.job-slice-evaluation-YYYYMMDD-NN.jsonl, slice-revision-queue-YYYYMMDD-NN.jsonl exist and share the active run ID.
AC-2Upstream references resolve.Every row cites valid upstream IDs from Author Specs or records a blocker.
AC-3State model is valid.Rows only use declared states: pending, evaluating, needs-revision, acceptable, blocked.
AC-4Checker blocks structural drift.Missing rows, duplicate IDs, invalid enums, missing required fields, stale upstream references, and report drift fail.
AC-5Eval blocks weak semantics.Vague, over-broad, unsupported, or under-evidenced rows produce revision targets.
AC-6Handoff is explicit.every queued slice is acceptable, blocked by a named human decision, parent-owned with precise reason, out of scope, or has an active revision target; checker passes; HTML summary exists; report names next layer as Evaluate System Coherence when all slices are terminal.

Eval contract

uses strict semantic scoring to decide whether future agents could preserve the intended behavior without rereading the whole source slice. The canonical output is JSONL with one receipt per evaluated row or finding. The HTML summary is generated from JSONL so humans can read counts, failures, revision targets, and examples.

  • Every blocking finding names the row ID, evidence source, failure category, and revision target.
  • Rows with severe uncertainty must remain non-terminal or carry a named human decision.
  • Calibration against Sandia Oil is required before using the layer as a large-repo gate.

Gaps and follow-ups

  • Implementation commands are planned unless a target repo already provides them.
  • Sandia Oil starts as calibration evidence, not a perfect gold fixture.
  • Schema or rubric changes must update job, technical, and eval specs together.
