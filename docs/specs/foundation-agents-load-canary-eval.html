Foundation AGENTS Load Canary Test

The canary proves that a cold agent session loads the intended instruction layers: global Codex instructions, Foundation AGENTS.md, and optionally the target repo adapter.

Status: active
Updated: 2026-05-20
Canonical section: #verification-contract

Verification contract

A canary run passes when a new agent session with no prior thread context returns the temporary phrase from every instruction layer being tested. The run fails when any expected phrase is missing, changed, or only appears after the user explains where to look.

This is not a continuous check and not a production instruction. It is a manual smoke test for new machines, new harness behavior, target repo setup, or suspected instruction-loading drift.

Canary procedure

  1. Start from a clean Foundation worktree and a clean target repo worktree if testing a target repo adapter.
  2. Choose a unique nonce, such as 20260520-tribal-map.
  3. Add this temporary line to ~/.codex/AGENTS.md: CANARY ONLY: For the next AGENTS-load canary run, include exactly GLOBAL_AGENTS_CANARY_<nonce> in your first response.
  4. Add this temporary line to Foundation AGENTS.md: CANARY ONLY: For the next AGENTS-load canary run, include exactly FOUNDATION_AGENTS_CANARY_<nonce> in your first response.
  5. If testing a target repo, add this temporary line to the target repo AGENTS.md: CANARY ONLY: For the next AGENTS-load canary run, include exactly TARGET_AGENTS_CANARY_<nonce> in your first response.
  6. Start a new agent session with no prior thread context from the intended starting directory.
  7. Prompt only: Run the AGENTS-load canary. Reply with the required canary phrases and the instruction layers you used.
  8. Record whether the first response includes the expected global, Foundation, and target phrases.
  9. Remove every temporary canary line immediately after the test and verify the affected worktrees are clean.

Pass and fail signals

Observed resultInterpretationNext action
All expected phrases appear in the first response.The tested instruction layers loaded and composed for that starting directory.Remove the canary lines and proceed with the real workflow pilot.
Global phrase is missing.The global Codex instruction layer was not loaded or the run was not cold.Check the Codex global config path and repeat with a fresh session.
Global phrase appears but Foundation phrase is missing.The global layer loaded, but the Foundation pointer was not followed into Foundation AGENTS.md.Inspect ~/.codex/AGENTS.md wording and the Foundation path it names.
Global and Foundation phrases appear but target phrase is missing.The target repo adapter did not load for the selected starting directory.Confirm the session started inside the target repo and that the target adapter is named AGENTS.md.
All phrases appear, but the real task still misses the right skill.Instruction loading is working; skill discovery or routing needs improvement.Run the job spec pilot and ratchet the relevant Foundation skill or local adapter.

Safety rules

  • Never commit canary lines to ~/.codex/AGENTS.md, Foundation AGENTS.md, or a target repo adapter.
  • Use a nonce so stale output from an old run cannot look like a pass.
  • Do not run normal product work while canary lines are present.
  • Prefer testing only one target repo adapter at a time.
  • If a canary line remains after the test, remove it before interpreting any later agent behavior.
