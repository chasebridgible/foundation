Operate Systems In The Real World

Shipped systems can be deployed, monitored, recovered, and improved from operating evidence.

Status: draft
Updated: 2026-06-06
Canonical section: #capability-intent

Capability intent

Shipped systems can be deployed, monitored, recovered, and improved from operating evidence.

This is a parent capability: it names an outcome family that decomposes into narrower child capabilities. It is not a phase, artifact group, department, or implementation layer.

Outcome contract

FieldContractEvidence or source
BeneficiaryHumans, agents, and software systems using Foundation.Foundation operating system goal.
Reliable outcomeShipped systems can be deployed, monitored, recovered, and improved from operating evidence.Capability remap plan.
Capability altitudeParent capability outcome family.foundation.capability-map-remap.job
Child capabilitiesSystems can ship safely; system health is visible; failure recovery is ready; operating lessons improve the system.System capability workshop and remap table.
Failure modeSystems exist as build artifacts but are not reliably deployed, monitored, recovered, or improved from reality.Remap eval failure model.

Parent capabilities decompose into narrower child capabilities. Jobs usually attach to those child capability specs unless this parent owns a direct operating job.
