Learn Agent Capabilities Over Time

Foundation needs a bounded way to notice meaningful changes in agent tooling, preserve those findings as durable JSON/JSONL substrate, promote durable lessons into principles-doc candidates, and notify the owner where review can happen.

Interfaces and sources

  • Allowed research scope is agent systems broadly: agent tooling, workflows, memory, evals, orchestration, Codex-style execution, and practical long-running-agent reliability.
  • Initial source families are OpenAI, Anthropic, Google, and Addy Osmani. Addy Osmani is expected to have a stable blog URL; the other families may need refined source entries over time.
  • Durable state lives in the Foundation repo as easy-to-query JSON or JSONL artifacts, following the same substrate pattern used by Foundation spec maps and backfill maps.
  • When a finding implies a durable agent principle, the scout should target the relevant principles document such as docs/principles/agent-principles.html or docs/principles/ai-evals-principles.html and open the change for review.
  • Codex automations provide wake-up and execution. Foundation GitHub state is the memory layer. GitHub comments are the preferred notification and review surface; email can be added as a secondary or fallback notification.
