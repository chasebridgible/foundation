Foundation Backfill Context Pack Technical Spec

This technical layer makes Foundation Backfill Context Pack deterministic enough for scripts and structured enough for agent judgment.

Status: active
Updated: 2026-06-07
Parent: foundation.backfill-context-pack.job

Implementation must preserve the source-of-record split: JSONL is canonical for layer rows and eval receipts, while HTML summaries are generated for human reading. Each command must be resumable from artifacts and must not rely on chat memory.

Contracts

ContractRuleReason
Upstream completenessThe layer starts only from a passing upstream handoff or an explicit blocker.Later synthesis must not hide earlier discovery gaps.
Canonical JSONLLayer rows and eval receipts are JSONL; HTML summaries are derived.Agents need durable machine state and users need readable review surfaces.
Bounded fill loopAgents process rows or slices in small batches and run checks after each batch.Long-running work survives context compaction and handoff.
Deterministic checkervalidates each Context Pack references valid queue slices and upstream row IDs, all required evidence categories have entries or explicit gaps, file-like evidence has exact paths plus line ranges, snippets, or symbols, generic full-file evidence is rejected, and pack size stays within configured limits.Structural drift should fail mechanically before semantic evaluation.
Semantic evalsamples packs against queue slices to judge whether necessary files or surfaces were omitted, irrelevant evidence is bloating context, specificity is weak, or the pack cannot support Process / Action Map and later Author Specs.Correctness includes meaning, specificity, and evidence quality.
Handoff gateeach active queued child/sole slice has a bounded Context Pack or explicit blocker; every pack preserves queue capability refs and rejects parent, needs-split, blocked, or non-queueable refs as active work; checker artifact passes for the current pack fingerprint; eval receipt and HTML summary exist for the current pack fingerprint; eval revision targets are zero; report names next layer as Process / Action Map.Each layer is a reproducible correct unit before the next layer consumes it.

Checker and fill loop

  1. Read active report, run log, upstream artifacts, and current layer artifact.
  2. Select next pending or failed batch by deterministic order plus risk priority.
  3. Inspect only the evidence needed for the batch and update canonical JSONL rows.
  4. Run the checker; repair structural failures before semantic eval.
  5. Run or append eval receipts; generate HTML summary from JSONL.
  6. Revise failed rows until the gate passes or a named human decision blocks progress.
