Visible Business Graph Evaluation

A graph is acceptable only when it is complete enough for humans and agents to inspect what the system can do, how work is handled, and why the graph claims are trusted.

Status: draft
Updated: 2026-06-06
Parent: foundation.visible-business-graph.system

Coverage plan

Unit tests cover parser and validator behavior, including required capability-to-job realized-by edges and job-to-capability supports edges. Integration tests cover Foundation-wide graph compatibility and example-client render/eval. Renderer tests execute the generated canvas script to verify centered top-down layout, visible parent/child capability rows, default capability hierarchy edges, centered initial pan in narrow canvases, shared document-navigation collapse behavior, deterministic branch accent color, selected-path edge policy, collapsed initial jobs, inline capability expansion, inline job expansion, job-context node visibility, actors-map expansion, aria-expanded updates, and pointer drag panning. Browser smoke review remains required before treating the canvas as a client-ready UI.

Gaps and follow-ups

V1 intentionally avoids authentication, collaborative editing, database-backed graph storage, and public client portals. Those require separate job and technical specs after the static canvas proves the substrate.
