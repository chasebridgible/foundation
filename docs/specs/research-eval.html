Research Workflow Eval

A research workflow is acceptable only when it is source-backed, scoped, deduped, access-readable, branch-reviewable, and honest about gaps.

Status: active
Updated: 2026-06-14
Canonical section: #verification-contract

Verification contract

The reviewer must be able to trust durable storage, merge/cleanup evidence, and the final share-facing access surface without reading chat history. The run passes only when official or source-backed evidence supports records, off-scope candidates are rejected or routed internally, changed artifacts are checked, access surfaces communicate cleanly, and publication evidence names either the PR/merge path or a blocker.

Co-evolution rule

Required: when a project changes its research workflow, update the project spec and checker. When Foundation changes this reusable pattern, update this job, technical spec, eval spec, and skill together.

If a future run fails because an agent followed the skill but produced unacceptable research, revise the skill and the target project's checker or eval rather than adding only a one-off instruction to the next prompt.
