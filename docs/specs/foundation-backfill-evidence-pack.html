Foundation Backfill Context Pack

Context Pack gathers only the files, surfaces, capabilities, tests, docs, schemas, and run state needed for a queued slice so work is resumable and context-bounded. This layer answers: What bounded evidence does the next slice need?

Status: active
Updated: 2026-05-29
Upstream: Define Spec Jobs
Next: Process / Action Map

This layer is a reproducible unit in the backfill substrate. It initializes from the previous layer, fills bounded batches, checks deterministic structure, writes canonical JSONL eval receipts, derives a readable HTML summary, revises failed rows, and records handoff state before the next layer begins.

GBrain's source-of-record lesson applies directly: canonical append-friendly artifacts own state; readable HTML summaries and report tables are derived views. Later agents cite row IDs and evidence links instead of relying on inherited conversation memory.

Small-loop journey

  1. Initialize: create the canonical Foundation Backfill Context Pack artifact from Define Spec Jobs handoff output.
  2. Select batch: choose pending rows or slices using deterministic order plus risk priority.
  3. Fill: agent reads the current queue slice, pulls upstream file/surface/capability IDs, selects exact evidence paths and snippets, names excluded near-misses, and records why the pack is sufficient for the slice.
  4. Check: run the deterministic checker and record result in the active report and run log.
  5. Evaluate: write canonical JSONL eval receipts and a readable HTML summary.
  6. Revise: route failed rows back to the fill loop with row IDs and findings.
  7. Handoff: advance only when the handoff gate passes or named human decisions block the layer.

Checker and eval contract

MechanismContract
Checkervalidates each Context Pack references valid queue slices and upstream row IDs, all required evidence categories have entries or explicit gaps, file-like evidence cites exact paths plus line ranges, snippets, or symbols, generic full-file evidence is rejected, and pack size stays within configured limits.
Eval receiptsamples packs against queue slices to judge whether necessary files or surfaces were omitted, irrelevant evidence is bloating context, evidence is too vague, or the pack cannot support Process / Action Map and later Author Specs. Eval output is canonical JSONL; HTML is a derived summary for human review.
Sandia Oil calibrationThe first real calibration target is Sandia Oil. Its output is not a gold benchmark unless a human-reviewed expected Artifact Inventory is created; it is used to catch obvious misses while context is tractable.

Handoff gate

each active queued slice has a bounded Context Pack or explicit blocker; checker artifact passes for the exact current Context Pack fingerprint; eval receipt and HTML summary exist for the exact current Context Pack fingerprint; all eval revision targets are resolved; report names next layer as Process / Action Map.

  • Canonical artifacts exist and share a run ID.
  • All rows are in valid terminal or explicitly blocked states.
  • Checker passes for structure, references, currentness, and required fields.
  • Eval JSONL and HTML summary exist for the current Context Pack artifact, with zero revision targets.
  • Active report records counts, artifact paths, checker/eval status, run-log sequence, and next layer.
