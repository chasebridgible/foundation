Foundation Backfill Capability Map Technical Spec

This technical layer makes Foundation Backfill Capability Map deterministic enough for scripts and structured enough for agent judgment.

Status: active
Updated: 2026-05-29
Parent: foundation.backfill-capability-map.job

Implementation must preserve the source-of-record split: JSONL is canonical for layer rows and eval receipts, while HTML summaries are generated for human reading. Each command must be resumable from artifacts and must not rely on chat memory. Capability rows are semantic groupings of reviewed Surface / Function Map rows, not generated path summaries.

Checker and fill loop

  1. Read active report, run log, upstream Surface / Function Map artifacts, and current Capability Map artifact.
  2. Select the next pending or failed ready surface with foundation:capability-map:fill -- --next.
  3. Review the selected Surface / Function Map row and enough nearby ready rows to decide actor/outcome grouping. Support classifications may inform evidence but do not become upstream capability rows.
  4. Write capability rows for that reviewed surface group with foundation:capability-map:fill -- --surface-ids ... --capabilities-json .... The command replaces prior rows overlapping the reviewed surface IDs and records upstream surface fingerprints.
  5. Run the checker; repair structural failures before semantic eval.
  6. Run or append eval receipts; generate HTML summary from JSONL.
  7. Revise every row named in eval revisionTargets, rerun eval, and record report state with nextLayer set to Define Spec Jobs only after check and eval pass with zero revision targets.
  8. Revise failed rows until the gate passes or a named human decision blocks progress.
