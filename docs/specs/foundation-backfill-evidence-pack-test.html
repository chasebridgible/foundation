Foundation Backfill Context Pack Test Spec

This test layer defines the deterministic and semantic gates that prove Foundation Backfill Context Pack can hand off safely.

Status: active
Updated: 2026-05-29
Parent: foundation.backfill-evidence-pack.descriptive

The tests treat the layer as a reproducible unit: structure is checked mechanically, semantic quality is judged through eval receipts, and the HTML summary is a derived review surface.

Acceptance mapping

IDConditionMust prove
AC-1Canonical artifacts exist.evidence-pack-YYYYMMDD-NN.jsonl, evidence-pack-eval-YYYYMMDD-NN.jsonl exist and share the active run ID.
AC-2Upstream references resolve.Every row cites valid upstream IDs from Define Spec Jobs or records a blocker.
AC-3State model is valid.Rows only use declared states: pending, packed, needs-evidence, ready-for-process-map; needs-evidence handoff rows require explicit blocker detail.
AC-4Checker blocks structural drift.Missing rows, duplicate IDs, invalid enums, missing required fields, stale upstream references, generic full-file evidence, stale checker artifacts, stale eval receipts, and report drift fail.
AC-5Eval blocks weak semantics.Vague, over-broad, unsupported, generic, oversized, or under-evidenced rows produce revision targets.
AC-6Handoff is explicit.each active queued slice has a bounded Context Pack or explicit blocker; checker artifact passes for the current pack fingerprint; eval receipt and HTML summary exist for the current pack fingerprint; revision targets are zero; report names next layer as Process / Action Map.

Eval contract

samples packs against queue slices to judge whether necessary files or surfaces were omitted, irrelevant evidence is bloating context, evidence is generic or full-file, or the pack cannot support Process / Action Map and later Author Specs. The canonical output is JSONL with one receipt per evaluated row or finding and the current pack fingerprint. The HTML summary is generated from JSONL so humans can read counts, failures, revision targets, and examples.

  • Every blocking finding names the row ID, evidence source, failure category, and revision target.
  • Every warning becomes a revision target; report handoff requires zero revision targets.
  • Rows with severe uncertainty must remain non-terminal or carry a named human decision.
  • Calibration against Sandia Oil is required before using the layer as a large-repo gate.

Gaps and follow-ups

  • Sandia Oil starts as calibration evidence, not a perfect gold fixture, and remains the planned large-repo calibration target.
  • Schema or rubric changes must update descriptive, technical, and test specs together.
