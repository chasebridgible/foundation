Foundation Workspace Doctor Eval Spec

The Foundation doctor test suite verifies that the setup diagnostic accepts a correctly connected Foundation workspace and rejects the target repo shapes that create drift.

Status: active
Updated: 2026-06-02
Canonical section: #verification-contract

Verification contract

The doctor is accepted when automated tests cover the core pass/fail contract and the command can run against the real Foundation repo without failing. Warnings are acceptable for target repo CI maturity when a target repo has not configured workflows yet.

The test suite should use temporary filesystem fixtures rather than mutating real target repos. The real command run supplies integration evidence for the current Foundation checkout.

Coverage plan

Automated tests cover deterministic filesystem and parser rules. They do not call GitHub, write target repos, or assert actual Codex instruction loading. The AGENTS-load canary requested for first-session confidence should be specified separately so it can intentionally create and remove a temporary instruction without polluting production agent rules.

Commands and evidence

  • npm run foundation:doctor:test runs the automated fixture suite.
  • npm run foundation:doctor runs the diagnostic against the current Foundation repo and global Codex pointer.
  • npm run foundation:doctor -- --repo /path/to/target-repo runs target repo adapter, namespace, and CI-shape checks.
  • npm run spec:check verifies the generated registry and metadata links after spec changes.

Gaps and follow-ups

  • AGENTS-load canary: define a separate smoke-test procedure that temporarily asks a new agent session to print a known phrase, proves the global Foundation pointer loaded, and then removes the canary instruction.
  • Target CI fixture: add a fixture or real target repo evidence for a pinned Foundation checkout once the first target repo workflow is created.
  • Adapter schema: replace duplicated-rule substring checks with a stricter adapter schema if target repo adapters become more varied.
