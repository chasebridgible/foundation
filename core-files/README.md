# Knowledge layer — core templates

Each markdown file is a **capture lane** for consulting intake: **why it matters**, then **questions** designed to surface how this specific company runs—not generic business theory.

Suggested order (adjust as needed):

1. `strategy.md` — direction, bets, constraints  
2. `value-proposition.md` — customer promise and proof  
3. `capabilities-products-services.md` — what you sell and deliver  
4. `competitive-profile.md` — alternatives and differentiation  
5. `financial-flows.md` — how money is earned and spent  
6. `organizational-structure.md` — roles, reporting, responsibilities  
7. `decision-making.md` — judgment, escalation, approvals  
8. `time-management.md` — where owner and key roles’ hours actually go  
9. `systems-and-coordination.md` — how work and information actually flow  
10. `processes-and-sops.md` — repeatable procedures vs tacit habit  
11. `tech-stack.md` — tools and integrations  
12. `data-and-metrics.md` — what gets measured and trusted  
13. `culture.md` — norms, incentives, “how things are done here”  
14. `goals.md` — business and personal horizons  

When documenting repeatable client or internal procedures, start from **`SOPs/sop-template.md`** and save finished SOPs alongside it or in a client-specific folder.

Define roles once in **`organization/org-structure-template.md`** (stable **Role IDs**, reporting lines, ownership). SOPs reference those IDs so responsibility stays consistent across docs.

Iterate templates after pilots; trim or merge lanes if interviews overlap.
