# Tech stack and tools

## Why this lane matters

Tools shape **what data exists**, **who can access it**, and **how fragile operations are**. This lane inventories platforms and integrations—not for audit theater, but to see where workflows break and where automation plugs in.

---

## Questions

1. **What tools run the business** today across CRM, scheduling, billing, accounting, payroll, communications, documents, inventory, field ops, support—and what sits in spreadsheets because nothing fits?

2. Which tools are **sources of truth** versus convenience—and where do two systems disagree?

3. What **integrations exist** (native or manual exports), and where does someone bridge gaps by copying data?

4. What **permissions and admin ownership** exist—who can grant access, reset passwords, pay invoices—and where is access brittle?

5. What tool choices were **recent migrations or regrets**—and what broke culturally when those changed?

6. Where does **technology failure** hurt fastest—internet outages, vendor downtime, lost files—and what manual backups exist?

7. What **capability do you wish you had** in tooling—better forecasting, routing, visibility, automation—that might already exist but hasn’t been prioritized?
